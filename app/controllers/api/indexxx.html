B. Usage

  • Basic usage data/users //Get all data data/user?id=1 //Detail for single data
  • Usage with limit data/users?limit=xxx xxx is count of limit.
  • Usage with sort data/users?limit=3&sort=[{"property":"id","dir":"asc"}]&format=json property is field name for sort. dir is direction for sorting, asc or desc.
  • Usage with filter data/users?filter=[{"property":"first_name","value":"super","param":"like"}]&format=json //1 filter property data/users?filter=[{"property":"first_name","value":"super","param":"like"},{"property":"first_name","value":"joko","param":"or_like"}]&format=json //2 filter properties property is field name.
    value is field value.
    param is SQL Where statements. If not written, use default "where property = value". Other param like, or_like, not_like, or_where, having
    operator is operator value. If not written, use default "where property = value".
